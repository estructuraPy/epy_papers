# epy_paper

Write a scientific paper **once**, export a **journal-compliant submission
draft** for any of 50 journals. Part of the ePy suite, sibling of
`epy_reports` (reports) and `epy_slides` (presentations).

The survey behind the catalog (see `REQUIREMENTS.md`) found that the published
two-column typeset is produced by the **publisher**, not the author — so the
author's job is the *submission manuscript*, and almost every journal's
manuscript is the same single-column, double-spaced, line-numbered template
with a few tunable parameters. epy_paper keeps the author's one source and
applies a per-journal **profile** to produce the compliant draft.

## Public API (one entry point, like the rest of the suite)

```python
from epy_paper import Paper, available_journals

available_journals()                      # [(id, name), ...]  — 50 journals
paper = Paper.from_file("manuscript.md")
result = paper.validate("eng-structures") # ValidationResult (typed warnings)
for w in result:
    print(w.code, w.severity, w.message)
paper.to_draft("eng-structures", "draft.docx")   # or fmt="tex" / "pdf"
```

## Authoring source (one file, written once)

A single Markdown file with a YAML front matter. Title / abstract / keywords
are plain scalars or bilingual `{es, en}` mappings; the journal profile decides
whether both languages are emitted. Highlights and declarations (CRediT,
competing interests, funding, data availability, AI disclosure) are modelled
too. Full spec in `REQUIREMENTS.md` (§6b) and worked examples in
`tests/fixtures/`.

## What ships

- `data/journals.json` — the journal catalog (**50 profiles**, 25 with a
  civil/structural focus): ASCE (Structural, Bridge, Composites,
  Geotechnical), Engineering Structures, JCSR, Structures, Thin-Walled
  Structures, Structural Safety, Computers & Structures, Soil Dynamics,
  Construction & Building Materials, Journal of Building Engineering, Cement
  and Concrete Research, ACI Structural/Materials, EESD, Bulletin of
  Earthquake Engineering, Earthquake Spectra, CACE, Materials and Structures
  (RILEM); the big multidisciplinary journals (Nature, Science, IEEE, ACS,
  PLOS, Lancet, NEJM, MDPI, JMLR, PRL); and Latin-American / Costa-Rican
  venues (Tecnología en Marcha, Kurú, Ingeniería UCR, Ingeniería e
  Investigación, DYNA, Acta Universitaria, IBRACON, Maderas, Revista de la
  Construcción) plus the wood-science targets from the replication set.
- `assets/csl/` — the bundled citation styles shared with epy_reports and
  epy_slides (IEEE, APA, Vancouver, Chicago, Harvard, MLA, ACS, AMA, Nature,
  Science, ASCE, Elsevier, Springer, …).
- `assets/latex/` — the official LaTeX classes: `elsarticle` (Elsevier),
  `IEEEtran` (IEEE), `ascelike` (ASCE), with their `.bst` styles. Journals
  with no bundled class fall back to a generic `article` template.
- `assets/templates/` — the universal Pandoc manuscript templates (LaTeX +
  HTML preview).
- `assets/reference_docx/submission.docx` — the Word manuscript reference doc
  (Times New Roman 12 pt, double spaced, US Letter), generated by
  `tools/make_reference_docx.py`.
- `REQUIREMENTS.md` — the requirements derived from the survey, plus the
  authoring format, rendering pipeline and validation contract.
- `docs/navarro_mora_publications.md` — real publications used as replication
  test fixtures (no relationship to the formats — validation targets only).

## Status

Catalog (50) + profiles + bilingual authoring format + structured validation +
per-profile content blocks + a Pandoc-backed draft exporter (DOCX/LaTeX/PDF)
with the journal's official LaTeX class and reference doc. Next: a Qt editor
reusing the epy_reports shell, and PyInstaller + Inno Setup packaging.
